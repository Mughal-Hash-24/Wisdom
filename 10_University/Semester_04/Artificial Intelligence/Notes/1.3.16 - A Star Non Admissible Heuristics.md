# Deep Dive: A* Search and the Consequence of Inadmissibility

## 1. Ontological Definition
A* search is an informed search algorithm that uses a total cost function $f(n) = g(n) + h(n)$ to navigate a state space. $g(n)$ represents the cost from the start to node $n$, while $h(n)$ is the estimated cost from $n$ to the goal. A heuristic is **admissible** if it never overestimates the cost ($h(n) \le h^*(n)$). When $h(n) > h^*(n)$, the heuristic is **inadmissible**. This violation of the "optimistic" bound fundamentally breaks the mathematical guarantee that A* will find the optimal (lowest-cost) path.

## 2. The Internal Mechanics (Under the Hood)
A* operates by maintaining a frontier (priority queue) of nodes ordered by their $f(n)$ values. The algorithm always expands the node with the lowest $f(n)$. 

Mathematically, the optimality of A* relies on the fact that if a node $n$ is on the optimal path, its $f(n)$ will eventually be recognized as lower than the $f$-value of any suboptimal goal.
If $h(n)$ overestimates:
1. Let $P^*$ be the optimal path with cost $C^*$.
2. Let $P_{sub}$ be a suboptimal path ending at goal $G_{sub}$ with cost $C_{sub} > C^*$.
3. If for some node $n$ on $P^*$, $h(n)$ is so high that $f(n) = g(n) + h(n) > C_{sub}$, A* will expand $G_{sub}$ and terminate before it ever expands $n$.

The "search tree" is pruned incorrectly because the overestimated heuristic makes the optimal path look "more expensive" than it actually is.

## 3. Systems Context & Anchoring (Analogy/C++)
**Analogy:** Imagine a pessimistic travel agent. You want the cheapest flight to Tokyo ($h^* = \$500$). The agent "overestimates" the cost of the direct flight, claiming it will cost $\$1200$. Meanwhile, they see a 3-stop flight for $\$800$. Because $\$800 < \$1200$, the agent books you on the longer, more expensive flight, failing to find the actual $\$500$ bargain.

**C++ Anchor:** In a typical C++ implementation using `std::priority_queue<Node>`, the ordering is determined by a custom comparator:
```cpp
struct CompareNode {
    bool operator()(Node* a, Node* b) {
        return (a->g + a->h) > (b->g + b->h); // Min-priority queue
    }
};
```
When `h` is inadmissible, the priority queue is essentially receiving "corrupted" data. The `top()` element is no longer guaranteed to be the prefix of an optimal solution, leading the `while(!pq.empty())` loop to a suboptimal termination.

## 4. Edge Cases & Constraints
*   **Speed vs. Accuracy:** Inadmissible heuristics often act like "Greedy Best-First Search." By overestimating, the algorithm becomes more "opinionated" and focused, which can significantly reduce the number of nodes expanded (faster execution) at the cost of the path quality.
*   **Weighted A*:** This is a deliberate use of overestimation ($f(n) = g(n) + \epsilon \cdot h(n)$ where $\epsilon > 1$). It is used in large state spaces (like robotics) to find a "good enough" solution quickly, with a guarantee that the cost is no more than $\epsilon$ times the optimal cost.
