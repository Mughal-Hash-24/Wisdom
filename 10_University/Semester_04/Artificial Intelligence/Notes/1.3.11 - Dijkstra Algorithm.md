# Algorithm Dissection: Dijkstra's Algorithm

## 1. The Logic (Operational Mechanics)
Dijkstra's algorithm solves the single-source shortest path problem for graphs with non-negative edge weights. It is the foundation of **Uniform Cost Search (UCS)** in AI.

### The "Relaxation" Primitive
The core operation is **Edge Relaxation**. For an edge $(u, v)$ with weight $w$:
If `dist[u] + w < dist[v]`:
   `dist[v] = dist[u] + w`
   `parent[v] = u`

## 2. Walkthrough Example
### The Graph
*   **Nodes:** A (Start), B, C, D (Goal)
*   **Edges:**
    *   (A, B): 4
    *   (A, C): 2
    *   (C, B): 1
    *   (B, D): 3
    *   (C, D): 10

### Step-by-Step Execution Trace

| Step | Current Node | Priority Queue (dist, node) | Distance Map | Visited |
| :--- | :--- | :--- | :--- | :--- |
| 0 | - | `[(0, A)]` | `{A:0, B:∞, C:∞, D:∞}` | `{}` |
| 1 | **A** | `[(2, C), (4, B)]` | `{A:0, B:4, C:2, D:∞}` | `{A}` |
| 2 | **C** | `[(3, B), (4, B), (12, D)]` | `{A:0, B:3, C:2, D:12}` | `{A, C}` |
| 3 | **B** | `[(6, D), (12, D)]` | `{A:0, B:3, C:2, D:6}` | `{A, C, B}` |
| 4 | **D** | `[]` | `{A:0, B:3, C:2, D:6}` | `{A, C, B, D}` |

### Detailed Analysis of Step 2:
When we are at **Node C**, we see an edge to **Node B** with weight 1. 
- Current `dist[B]` is 4 (from A).
- New potential `dist[B]` is `dist[C] + 1` = $2 + 1 = 3$.
- Since $3 < 4$, we **relax** the edge. Node B is re-added to the Priority Queue with the lower cost.

## 3. Implementation (High-Performance)
```python
import heapq

def dijkstra(graph, start):
    distances = {node: float('inf') for node in graph}
    distances[start] = 0
    pq = [(0, start)] # (cost, node)

    while pq:
        current_dist, u = heapq.heappop(pq)

        if current_dist > distances[u]:
            continue

        for v, weight in graph[u].items():
            distance = current_dist + weight
            if distance < distances[v]:
                distances[v] = distance
                heapq.heappush(pq, (distance, v))
    
    return distances
```

## 4. Edge Cases (The Inversionist)
- **Negative Weights:** Dijkstra fails. It assumes that once a node is popped from the PQ, its shortest path is found. Negative edges can invalidate this assumption later.
- **Disconnected Graphs:** Nodes unreachable from the start will retain `float('inf')` distance.
- **Cycles:** Dijkstra handles positive cycles naturally by updating only when a strictly better path is found.
