# Technical breakdown: Search Implementation Logic

## 1. Surgical Definition (Internals)
The "meticulousness" or "greediness" of a search algorithm is entirely determined by the **Priority Key** used in its Fringe (Frontier) data structure.

| Algorithm | Data Structure | Priority Key ($k$) | Behavioral Result |
| :--- | :--- | :--- | :--- |
| **BFS** | FIFO Queue | None (Order of Discovery) | Explores in circular layers of depth. |
| **UCS** | Priority Queue | $k = g(n)$ (Path Cost) | **Meticulous:** Always expands the cheapest known path first. |
| **Greedy** | Priority Queue | $k = h(n)$ (Heuristic) | **Visionary:** Always expands the path that "looks" closest to the goal. |

## 2. The Laboratory (Proof of Concept)
The following Python code demonstrates how a single search framework becomes either UCS or Greedy simply by changing one variable in the `heapq.heappush` call.

```python
import heapq

# SEARCH ENGINE CORE
def search_engine(start, goal, graph, heuristic, mode="UCS"):
    # (Priority, Node)
    frontier = [(0, start)]
    visited = set()

    while frontier:
        priority, current = heapq.heappop(frontier)
        
        if current == goal: return "Success"
        visited.add(current)

        for neighbor, step_cost in graph[current].items():
            if neighbor not in visited:
                # THE LOGIC SWITCH
                if mode == "UCS":
                    # Meticulous: Path cost is the key
                    new_priority = priority + step_cost 
                else: # mode == "Greedy"
                    # Visionary: Heuristic is the key
                    new_priority = heuristic[neighbor]
                
                heapq.heappush(frontier, (new_priority, neighbor))
```

## 3. Memory & System Context
*   **UCS (The Turtle):**
    *   **Overhead:** High. It may store a massive number of nodes in the priority queue because it refuses to "jump ahead" if there's a cheaper path nearby.
    *   **Heap Property:** $g(n)$ is monotonically increasing. The heap stays balanced as we deepen the search.
*   **Greedy (The Rabbit):**
    *   **Overhead:** Low (ideally). It heads straight for the goal, ignoring large swaths of the graph.
    *   **Risk:** If the heuristic is misleading, the heap expands down a "dead end" branch deeply, wasting memory on a path that might be infinite or extremely costly.

## 4. Best Practices & Anti-Patterns
- **UCS Anti-Pattern:** Using UCS on a graph with very small edge weights but a massive branching factor. The queue will explode before the goal is reached.
- **Greedy Anti-Pattern:** Using Greedy when path cost matters. Greedy is NOT optimal; it will take a $1,000,000$ cost path if the heuristic says the end is only 1 inch away.
- **The Ideal:** Use **A*** ($g(n) + h(n)$) to balance the meticulousness of the queue with the focus of the priority key.
