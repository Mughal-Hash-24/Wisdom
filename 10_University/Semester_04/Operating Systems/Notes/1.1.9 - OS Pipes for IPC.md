# Technical breakdown: Pipes in Inter-Process Communication (IPC)

## 1. Surgical Definition (Internals)
At the kernel level, a **pipe** is a unidirectional FIFO (First-In-First-Out) byte stream implemented as a circular buffer in kernel memory. It solves the problem of **synchronized data exchange** between processes without the overhead of shared memory or the complexity of sockets.
- **Anonymous Pipes:** Transient channels created via `pipe()`. They lack a name in the filesystem and are restricted to related processes (e.g., parent-child) because child processes inherit the file descriptors (FDs).
- **Named Pipes (FIFOs):** Persistent filesystem objects created via `mkfifo()`. They allow unrelated processes to communicate by opening the FIFO path like a file.
The kernel manages synchronization: if the pipe is full, the writer blocks; if the pipe is empty, the reader blocks.

## 2. The Laboratory (Proof of Concept)
This C++ showcase implements an anonymous pipe between a parent (writer) and child (reader).

```cpp
#include <iostream>
#include <unistd.h>
#include <sys/wait.h>
#include <cstring>
#include <fcntl.h>

int main() {
    int pipefd[2]; 
    if (pipe(pipefd) == -1) return 1;

    pid_t pid = fork();
    if (pid == 0) {         // Child Process (Reader)
        close(pipefd[1]);   // 1. Close unused write end
        char buffer[128];
        ssize_t bytes = read(pipefd[0], buffer, sizeof(buffer));
        if (bytes > 0) {
            buffer[bytes] = '\0';
            std::cout << "[Reader] Received: " << buffer << std::endl;
        }
        close(pipefd[0]);
        return 0;
    } else {                // Parent Process (Writer)
        close(pipefd[0]);   // 2. Close unused read end
        const char* msg = "Pipes bypass disk I/O.";
        write(pipefd[1], msg, strlen(msg)); // 3. Kernel copy
        close(pipefd[1]);   // 4. Closing triggers EOF for reader
        wait(NULL);
    }
    return 0;
}
```

## 3. Memory & System Context
* **Stack/Heap:** Pipe buffers (typically 64KB) live in **Kernel Space**. Data is copied from the writer’s buffer to the kernel, then to the reader’s buffer.
* **Cost:** Highly efficient. No context switching is required for the data itself, only the syscalls (`read`/`write`).
* **Life-cycle:** Anonymous pipes vanish once all FDs are closed. Named pipes persist on disk but their *data* is volatile and exists only while processes are connected.

## 4. Best Practices & Anti-Patterns
* **The "Broken Pipe" (SIGPIPE):** Writing to a pipe with no active readers triggers a `SIGPIPE` signal. If unhandled, the process crashes.
* **Partial Reads/Writes:** For pipes, `write()` is atomic only if the size is $\leq$ `PIPE_BUF` (4096 bytes on Linux). 
* **Deadlocks:** If both processes wait to read or write without closing FDs correctly, the system hangs. Always close unused ends immediately after `fork()`.
