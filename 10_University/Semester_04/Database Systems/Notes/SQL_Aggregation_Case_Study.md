## 1. Case Study: The "TechStore" Analytics

> **Prompt:** "Define a detailed case study that forces the use of the 5 aggregate functions of SQL and then provide the solution of each part. There must be 6 parts. 1 for each and 6th one should implement each of the functions"
> **Lens Applied:** The Architect / The Pragmatic Developer

### Scenario
You are the Database Analyst for "TechStore". You have a table `Sales` with columns: `SaleID`, `Product`, `Category`, `Amount`, `SaleDate`.

### Part 1: Traffic Analysis (COUNT)
**Question:** How many individual sales transactions have occurred?
```sql
SELECT COUNT(*) AS Total_Transactions FROM Sales;
```

### Part 2: Revenue Analysis (SUM)
**Question:** What is the total revenue generated by the store?
```sql
SELECT SUM(Amount) AS Total_Revenue FROM Sales;
```

### Part 3: Peak Performance (MAX)
**Question:** What was the most expensive item sold (highest sale amount)?
```sql
SELECT MAX(Amount) AS Highest_Ticket_Item FROM Sales;
```

### Part 4: Entry Level Analysis (MIN)
**Question:** What was the cheapest item sold, and when was the first sale ever made?
```sql
SELECT MIN(Amount) AS Cheapest_Item, MIN(SaleDate) AS First_Sale_Day 
FROM Sales;
```

### Part 5: Average Basket Size (AVG)
**Question:** What is the average value of a customer's purchase?
```sql
SELECT AVG(Amount) AS Average_Transaction_Value FROM Sales;
```

### Part 6: The Executive Dashboard (Combined)
**Question:** Generate a single report showing Total Sales count, Total Revenue, Min, Max, and Average sale amount.
```sql
SELECT 
    COUNT(*) AS "Total Sales",
    SUM(Amount) AS "Total Revenue",
    MIN(Amount) AS "Lowest Sale",
    MAX(Amount) AS "Highest Sale",
    AVG(Amount) AS "Average Sale"
FROM Sales;
```

## 2. Practice Problem: Clerk Salaries

> **Prompt:** "Write an SQL code that sums the total salary being given to all the employees of type clerk"
> **Lens Applied:** The Optimizationist

### Solution
```sql
SELECT SUM(salary) AS "Total Clerk Payroll"
FROM employees
WHERE job = 'CLERK'; /* Or job LIKE 'CLERK' */
```

### Breakdown
1.  **Filter First:** The database engine first applies the `WHERE job = 'CLERK'` filter to isolate only the relevant rows.
2.  **Aggregate Second:** It then passes the `salary` values of these filtered rows to the `SUM` accumulator.
3.  **Efficiency:** This is much faster than summing everything and filtering later (which isn't even possible in standard SQL without subqueries).
