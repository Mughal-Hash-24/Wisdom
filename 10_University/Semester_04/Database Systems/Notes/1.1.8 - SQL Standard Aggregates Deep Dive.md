## 1. The COUNT Function

> **Prompt:** "Explain in detail the aggregate function of count with all possible cases and syntax and edge cases and default behaviours with detailed examples of each and make sure the concept is detailed like textbooks and the code examples are explained in depth. Numeric + Non-numeric"
> **Lens Applied:** The Algorithmist

### Definition
Returns the cardinality (number of items) of a set.

### Variants & Syntax
1.  **`COUNT(*)`**: Counts **all rows**, including NULLs and duplicates. It measures the "physical size" of the table or group.
    ```sql
    SELECT COUNT(*) FROM employees; -- Returns total staff count (e.g., 100)
    ```
2.  **`COUNT(column)`**: Counts only **Non-NULL** values in that specific column.
    ```sql
    SELECT COUNT(commission) FROM employees; 
    -- If 20 people have NULL commission, this returns 80.
    ```
3.  **`COUNT(DISTINCT column)`**: Counts unique, Non-NULL values.
    ```sql
    SELECT COUNT(DISTINCT job_title) FROM employees; 
    -- Returns how many unique roles exist (e.g., 5).
    ```

### Edge Cases
*   **All NULLs:** `COUNT(null_column)` returns 0.
*   **Non-Numeric:** Works on Strings, Dates, etc. `COUNT(name)` counts employees with names.

## 2. The MAX Function

> **Prompt:** "Explain in detail the aggregate function of max..."
> **Lens Applied:** The Algorithmist

### Definition
Returns the maximum value in a set.

### Behavior by Data Type
1.  **Numeric:** Returns the largest number.
    ```sql
    SELECT MAX(salary) FROM employees; -- Highest paid employee
    ```
2.  **String (Varchar):** Returns the last value lexicographically (Alphabetical Z-A).
    ```sql
    SELECT MAX(name) FROM employees; -- "Zack" comes after "Alice"
    ```
3.  **Date/Time:** Returns the **latest** date (most recent).
    ```sql
    SELECT MAX(hire_date) FROM employees; -- The junior-most employee
    ```

## 3. The SUM Function

> **Prompt:** "Explain in detail the aggregate function of sum..."
> **Lens Applied:** The Algorithmist

### Definition
Returns the summation of all values in a set.

### Constraints
*   **Numeric Only:** Cannot `SUM` strings or dates. `SUM('A', 'B')` throws an error.
*   **NULL Handling:** `NULL` values are treated as 0 (ignored). `10 + NULL + 20` in scalar math is `NULL`, but in Aggregate `SUM`, it is `30`.

### Example
```sql
SELECT SUM(salary) FROM employees WHERE dept_id = 10;
-- Returns total payroll for Department 10.
```

## 4. The AVG Function

> **Prompt:** "Explain in detail the aggregate function of avg..."
> **Lens Applied:** The Algorithmist

### Definition
Returns the Arithmetic Mean. Internally computed as `SUM(col) / COUNT(col)`.

### The NULL Trap (Critical Edge Case)
`AVG` ignores NULLs completely. It does **not** treat them as zero.
*   **Data:** `{100, 200, NULL}`
*   **Calculation:** $(100 + 200) / 2 = 150$. (Divides by 2, not 3).
*   **If you want to treat NULL as 0:** You must use `COALESCE` or `NVL`.
    ```sql
    SELECT AVG(COALESCE(commission, 0)) FROM employees;
    -- Now calculates (100 + 200 + 0) / 3 = 100.
    ```

## 5. The MIN Function

> **Prompt:** "Explain in detail the aggregate function of min..."
> **Lens Applied:** The Algorithmist

### Definition
Returns the minimum value in a set.

### Behavior by Data Type
1.  **Numeric:** Returns the smallest number.
    ```sql
    SELECT MIN(salary) FROM employees; -- Lowest paid entry
    ```
2.  **String:** Returns the first value lexicographically (Alphabetical A-Z).
    ```sql
    SELECT MIN(name) FROM employees; -- "Aaron"
    ```
3.  **Date/Time:** Returns the **earliest** date (oldest).
    ```sql
    SELECT MIN(hire_date) FROM employees; -- The longest-serving employee
    ```
