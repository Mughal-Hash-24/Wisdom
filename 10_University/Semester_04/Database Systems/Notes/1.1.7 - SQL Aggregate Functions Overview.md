# Deep Dive: SQL Aggregate Functions

### 1. The Core Concept (Many to One)
In the relational model, standard queries operate row-by-row (Scalar). **Aggregate Functions** operate on a **set** of rows (a bag of tuples) to produce a **single** summary value.
*   **Input:** A column (vector) of values from $N$ rows.
*   **Output:** A scalar value (1 row).
*   **Mathematical Notation:** $f( \{x_1, x_2, ..., x_n\} ) = y$

### 2. The Big Five (Standard Aggregates)
These are the standard functions supported by almost every SQL engine (ANSI SQL).

| Function | Logical Operation | Purpose | Real World Analogy |
| :--- | :--- | :--- | :--- |
| **COUNT()** | Cardinality | Counts the number of rows or non-null values. | A turnstile counting people entering a stadium. |
| **SUM()** | Addition | Adds up all values in a numeric column. | A cash register tallying the day's receipts. |
| **AVG()** | Arithmetic Mean | Calculates the mean ($\frac{\sum x}{n}$). | A teacher calculating the class average test score. |
| **MAX()** | Upper Bound | Finds the largest/latest value. | A thermostat recording the peak temperature of the day. |
| **MIN()** | Lower Bound | Finds the smallest/earliest value. | A runner recording their fastest lap time. |

### 3. Operational Constraints
*   **NULL Handling:** Generally, aggregate functions **ignore NULLs**, except for `COUNT(*)`.
*   **WHERE Clause:** Aggregates cannot be used in the `WHERE` clause (e.g., `WHERE SUM(cost) > 100` is illegal). You must use the `HAVING` clause for filtering based on aggregates.
*   **Data Types:** `SUM` and `AVG` require numeric data. `MIN`, `MAX`, and `COUNT` work on almost any data type (Numbers, Strings, Dates).
